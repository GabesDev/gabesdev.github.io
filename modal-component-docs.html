Modal HTML Component

The modal HTML component displays focused content in an overlay above the current page. It is useful for confirmations, lightweight forms, announcements, and any interaction that should temporarily capture attention without leaving the current view.

When to use it: Use the modal for short, self-contained tasks. Avoid placing long multi-step flows inside a modal when a dedicated page would be clearer.

Key features

  • Centered overlay layout with a visually separated dialog surface
  • Header, content, and actions regions for predictable structure
  • Backdrop support for dimming background content
  • Optional dismiss and primary action buttons

Recommended behavior

  • Open the modal from a clear trigger such as a button or link
  • Move keyboard focus into the modal when it opens
  • Return focus to the triggering element when it closes
  • Support closing via an explicit close action and the Escape key

Example markup

<div class="modal-backdrop" hidden>
  <div class="modal" role="dialog" aria-modal="true" aria-labelledby="modal-title">
    <div class="modal__header">
      <h2 id="modal-title">Delete item</h2>
      <button type="button" aria-label="Close modal">×</button>
    </div>

    <div class="modal__body">
      <p>Are you sure you want to delete this item? This action cannot be undone.</p>
    </div>

    <div class="modal__actions">
      <button type="button">Cancel</button>
      <button type="button">Delete</button>
    </div>
  </div>
</div>

API overview

Part Purpose
.modal-backdrop Provides the full-screen overlay behind the dialog.
.modal Wraps the visible dialog surface and main content.
.modal__header Contains the title and optional close action.
.modal__body Holds descriptive text, forms, or other focused content.
.modal__actions Groups confirmation and dismissal buttons.
